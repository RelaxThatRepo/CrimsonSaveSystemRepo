Saving World Data (C++)

This guide covers the C++ implementation for saving the state of Actors placed in your level (chests, doors, etc.). This process uses the ICrimsonSaveableActor interface.

This guide is for C++ programmers. For a Blueprint-only approach, see the Blueprint guide.

Step 1: Create Your Actor Data Object (Optional)

The manager stores a UObject* payload for each actor. It's good practice to create a simple UObject class to hold this data.

// MySaveableChestData.h
#pragma once
#include "MySaveableChestData.generated.h"

UCLASS()
class UMySaveableChestData : public UObject
{
    GENERATED_BODY()
public:
    UPROPERTY()
    bool bHasBeenOpened;
};

Step 2: Implement the Interface

In your Actor class (e.g., AMySaveableChest), add the ICrimsonSaveableActor interface.

// MySaveableChest.h
#pragma once
#include "ICrimsonSaveableActor.h"
#include "MySaveableChest.generated.h"

UCLASS()
class AMySaveableChest : public AActor, public ICrimsonSaveableActor
{
    GENERATED_BODY()
    
public:
    // --- ICrimsonSaveableActor Interface ---
    virtual FGuid GetSaveID_Implementation() const override;
    virtual void ResetToDefaultState_Implementation() override;
    virtual void RestoreFromSaveData_Implementation(UObject* ActorData) override;
    // --- End Interface ---
    
    // The actor's live property
    UPROPERTY(VisibleAnywhere, Replicated)
    bool bIsOpen = false;

    // The actor's unique ID for saving
    UPROPERTY(EditAnywhere, BlueprintReadOnly, Category = "Save System")
    FGuid SaveID;
};

Step 3: Implement Interface Functions

In your .cpp file, provide logic for the interface functions.

CRITICAL: The FGuid SaveID must be unique for every saveable actor *in the level*.

// MySaveableChest.cpp
#include "MySaveableChestData.h"

FGuid AMySaveableChest::GetSaveID_Implementation() const
{
    return SaveID;
}

void AMySaveableChest::RestoreFromSaveData_Implementation(UObject* ActorData)
{
    if (UMySaveableChestData* ChestData = Cast<UMySaveableChestData>(ActorData))
    {
        this->bIsOpen = ChestData->bHasBeenOpened;
        
        // ... (update mesh, disable interaction, etc.)
    }
   
}

Step 4: Notify the Manager of Changes

Your actor *must* tell the WorldManager when its state changes. The manager doesn't poll; it relies on notifications.

// MySaveableChest.cpp
#include "CrimsonSaveWorldManagerSubsystem.h"
#include "MySaveableChestData.h"

void AMySaveableChest::OnInteract_OpenChest()
{
    if (bIsOpen) return; // Already open
    
    bIsOpen = true;
    // ... (play animation, spawn loot)

    // --- NOTIFY THE SAVE SYSTEM ---
    if (UCrimsonSaveWorldManagerSubsystem* WorldSaveManager = GetWorld()->GetSubsystem<UCrimsonSaveWorldManagerSubsystem>())
    {
        // Create the payload object
        UMySaveableChestData* ChestData = NewObject<UMySaveableChestData>();
        ChestData->bHasBeenOpened = true;
        
        // Send the new state to the manager
        WorldSaveManager->OnActorStateChanged(this, ChestData);
    }
}

If your actor is permanently destroyed (like a breakable wall), you should call OnActorDestroyed instead.

// MyBreakableWall.cpp
#include "CrimsonSaveWorldManagerSubsystem.h"

void AMyBreakableWall::OnTakeDamage_Destroy()
{
    // --- NOTIFY THE SAVE SYSTEM ---
    if (UCrimsonSaveWorldManagerSubsystem* WorldSaveManager = GetWorld()->GetSubsystem<UCrimsonSaveWorldManagerSubsystem>())
    {
        // Pass 'nullptr' as the payload, we just care that it's destroyed
        WorldSaveManager->OnActorDestroyed(this, nullptr);
    }
    
    // ... (spawn effects, etc.)
    Destroy();
}
