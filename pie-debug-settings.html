PIE & Debug Settings

The system's debugging and Play-In-Editor (PIE) behavior is controlled by UCrimsonSaveGameManagerDeveloperSettings. You can find these settings in your project's Project Settings > Game > Crimson Save Game Manager.

Player Manager (PIE) Settings

These settings control the UCrimsonSaveGameManagerSubsystem (GameInstance) during PIE sessions.

  • Default PIE Save Slot: The save slot index (e.g., 0) to automatically load or create when you start a PIE session.
  • Default PIE Character Name: The character name used (e.g., "Play In Editor Character") if a new debug save file needs to be created.
  • Reset PIE Save Slot On Start: If true, the DefaultPIESaveSlot will be completely deleted and reset every time you press "Play". This is perfect for testing your "New Game" flow repeatedly.
  • Main Menu Map: This is a crucial setting. If you set your project's main menu map here, the subsystem will *bypass* all the PIE debug logic if it detects it's loading that map. This allows you to start PIE in your main menu and test the save/load system as a real player would.

World Manager Settings

These settings control the UCrimsonSaveWorldManagerSubsystem (World).

  • Auto Load World State: If true, the world subsystem will automatically call LoadWorldState() during its OnWorldBeginPlay event. This is the recommended setting for most games. If false, you must call LoadWorldState() manually.
  • Save World State After Every Change: If true, the world subsystem will call SaveWorldState() *every single time* an actor calls OnActorStateChanged or OnActorDestroyed.

Performance Warning: bSaveWorldStateAfterEveryChange is excellent for debugging to ensure your actors are saving correctly, but it can cause significant performance hits and disk I/O. It is strongly recommended to disable this in a packaged build.
