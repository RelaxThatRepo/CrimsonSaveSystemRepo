Saving World Data (Blueprints)

This guide covers the Blueprint-only implementation for saving the state of Actors placed in your level (chests, doors, etc.). This process uses the ICrimsonSaveableActor interface.

This guide is for Blueprints. For a C++ approach, see the C++ guide.

▶️ Saving World Data

Step 1: Create Your Actor Data Struct (Optional)

The manager stores an FInstancedStruct payload for each actor. It's good practice to create a simple Struct Blueprint to hold this data.

  1. Right-click in the Content Browser -> Blueprints -> Structure.
  2. Name it Struct_WorldActorData.
  3. Open it and add a variable Scale of type Vector.

Step 2: Implement the Interface

In your Actor Blueprint (e.g., SaveGame_WorldActor), add the ICrimsonSaveableActor interface.

  1. Go to Class Settings in the toolbar.
  2. In the Details panel, find the "Interfaces" section.
  3. Click "Add" and search for ICrimsonSaveableActor. Select it.
  4. Add a new variable named SaveID of type Guid.
  5. Click the "eye" icon to make it Instance Editable.

CRITICAL: After placing SaveGame_WorldActor in your level, you must select it, go to the Details panel, and click the "Generate new GUID" button next to your SaveID variable. Each instance *must* have a unique ID.

Step 3: Implement Interface Functions

After adding the interface, new functions will appear in the "My Blueprint" panel. Implement them.

GetSaveID

Double-click this function. Drag in your SaveID variable and connect it to the Return Value.

RestoreFromSaveData

This function applies the saved data to your actor. The Actor Data input is now an Instanced Struct.

  1. Drag off the Actor Data input pin and use the Get Instanced Struct Value node.
  2. Drag from the Value on this node and break it into its correct type, in this case Struct_WorldActorData.
  3. Set your actor's live variable (e.g., Scale) with this saved value.
  4. Call any events needed to visually update the actor (e.g., play the "scaling" animation).

Step 4: Notify the Manager of Changes

Your actor *must* tell the WorldManager when its state changes. The manager doesn't poll; it relies on notifications.

In your OnInteract or similar event:

  1. Get the CrimsonSaveWorldManagerSubsystem.
  2. Use the Make Instanced Struct node. Set Value by making a Struct_WorldActorData and setting is current scale value.
  3. Call OnActorStateChanged on the subsystem. Pass self for "In Actor" and the return value from Make Instanced Struct for "Actor Data".

If your actor is destroyed, call OnActorDestroyed instead, passing self and an optional Instanced Struct for any final state.
